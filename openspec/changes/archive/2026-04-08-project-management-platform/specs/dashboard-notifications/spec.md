## ADDED Requirements

### Requirement: Dashboard 项目概览
系统 SHALL 在 Dashboard 页面展示所有项目的关键指标，包含需求完成率、缺陷数量、当前迭代进度和待处理事项数。

#### Scenario: 加载 Dashboard
- **WHEN** 用户进入 Dashboard 页面
- **THEN** 系统返回当前用户有权查看的项目概览数据，包含各项目的需求总数、完成数、缺陷数和当前迭代名称

#### Scenario: 查看单项目指标
- **WHEN** 用户在 Dashboard 选择某个项目
- **THEN** 系统展示该项目的详细指标（需求分布饼图、迭代燃尽图、缺陷趋势图）

### Requirement: 关键指标展示
系统 SHALL 展示以下关键指标：需求生命周期分布（各状态数量）、测试通过率、缺陷严重程度分布、个人待办事项。

#### Scenario: 需求状态分布
- **WHEN** 用户查看项目 Dashboard
- **THEN** 系统展示当前项目各需求状态的数量分布（待评审 N 个，待技评 N 个，...）

### Requirement: 通知系统
系统 SHALL 在重要事件发生时（需求状态流转、缺陷分配、@提及等）向相关人员发送通知，支持企业微信群机器人 Webhook 推送。

#### Scenario: 需求状态变更通知
- **WHEN** 需求状态发生流转（如从"开发中"变为"待测试"）
- **THEN** 系统通过企业微信 Webhook 向测试负责人发送通知，包含需求名称、状态变更详情和链接

#### Scenario: 缺陷分配通知
- **WHEN** 缺陷被分配给某个开发人员
- **THEN** 系统向该开发人员发送企业微信通知，包含缺陷标题、严重程度和链接

### Requirement: 站内通知中心
系统 SHALL 在平台顶部导航栏展示未读通知数量，用户可查看历史通知列表。

#### Scenario: 查看通知列表
- **WHEN** 用户点击顶部通知图标
- **THEN** 系统展示最近 50 条通知，按时间倒序，显示已读/未读状态

#### Scenario: 标记通知已读
- **WHEN** 用户点击某条通知或点击"全部已读"
- **THEN** 系统更新通知状态，未读数量更新
