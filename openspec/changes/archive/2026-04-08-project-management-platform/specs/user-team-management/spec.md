## ADDED Requirements

### Requirement: 用户 CRUD
系统 SHALL 支持用户的创建、查询、更新和删除，用户信息包含用户名、密码（加密存储）、邮箱、真实姓名、角色和企业微信信息。

#### Scenario: 创建用户
- **WHEN** 管理员提交用户信息至 `POST /api/users`
- **THEN** 系统创建用户，密码使用 bcrypt 加密存储，返回 HTTP 201

#### Scenario: 获取用户列表
- **WHEN** 管理员请求 `GET /api/users`
- **THEN** 系统返回所有用户列表（不含密码字段），支持按角色/姓名筛选

#### Scenario: 非管理员无法创建用户
- **WHEN** 普通用户请求 `POST /api/users`
- **THEN** 系统返回 HTTP 403

### Requirement: 角色与权限分配
系统 SHALL 支持六种预定义角色（产品经理、开发人员、测试人员、项目经理、管理员、超级管理员），每种角色有对应的操作权限集合。

#### Scenario: 修改用户角色
- **WHEN** 管理员请求 `PUT /api/users/:id`，更新 `role` 字段
- **THEN** 系统更新用户角色，该用户下次请求时权限立即生效，返回 HTTP 200

#### Scenario: 超级管理员可执行所有操作
- **WHEN** 超级管理员角色用户访问任意 API
- **THEN** 系统不拒绝任何请求

### Requirement: 团队 CRUD
系统 SHALL 支持团队的创建、查询、更新和删除，团队可关联多个用户成员。

#### Scenario: 创建团队
- **WHEN** 项目经理提交团队名称和成员列表至创建团队接口
- **THEN** 系统创建团队，建立用户与团队的关联，返回 HTTP 201

#### Scenario: 添加/移除团队成员
- **WHEN** 团队负责人更新团队成员列表
- **THEN** 系统更新成员关联关系，返回 HTTP 200

### Requirement: 工作量统计
系统 SHALL 统计每个用户负责的需求数量和缺陷数量，支持按时间范围和项目筛选。

#### Scenario: 查询个人工作量
- **WHEN** 用户或管理员请求工作量统计接口
- **THEN** 系统返回指定用户在指定时间范围内负责的需求数、完成需求数、缺陷数等指标

#### Scenario: 查询团队工作量
- **WHEN** 项目经理请求团队工作量统计
- **THEN** 系统返回团队成员的工作量汇总，支持按成员下钻查看
